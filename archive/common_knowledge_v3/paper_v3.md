# A Deterministic Solution to the Infinite Regress of Common Knowledge

## Abstract

This paper resolves the infinite regress of common knowledge in the Stag Hunt coordination game by introducing the Sacred File and State Vector architecture under the Aumann (1976) partitional information model. We prove that this deterministic mechanism terminates the regress at step one, ensuring common knowledge of the payoff-dominant equilibrium (Stag, Stag) with payoffs (2, 2) after a single shared read. Unlike prior approaches that falter under incomplete information or asynchronous constraints, our architecture collapses private information partitions to a shared public element, verifiable via a public audit log. Applied to the Stag Hunt, it guarantees coordination on (Stag, Stag), eliminating failure due to belief uncertainty. Boundary analysis reveals fragility to assumption violations like partial reads or observability failures, defining precise limits of applicability. This result reframes coordination failure as an information structure problem, solvable through formal design, with implications for game theory and mechanism design.

## Definitions Block: Stag Hunt Coordination Game under Aumann (1976) Partitional Information Model

This section establishes the formal foundation for analyzing common knowledge in the **Stag Hunt** coordination game using the **Aumann (1976) partitional information model** as the primary framework. All symbols and objects used in this paper are defined here, with standard concepts presented concisely and novel contributions elaborated with full rigor.

**Definition 1 (State Space).** Let Ω be the set of all possible states of the world, representing the complete description of the environment and actions in the Stag Hunt game. Each ω ∈ Ω encodes the strategies chosen by the agents (Stag or Hare) and the information available to them at a given time [Aumann 1976].

**Definition 2 (Agent Set).** Let N = {1, 2} be the set of two rational agents participating in the Stag Hunt game. Each agent i ∈ N chooses a strategy based on their information about the state of the world [Skyrms 2004].

**Definition 3 (Stag Hunt Payoff Matrix).** The Stag Hunt is a two-player coordination game with the following payoff matrix, where rows represent agent 1's choices and columns represent agent 2's choices: (Stag, Stag) = (2, 2), (Hare, Hare) = (1, 1), (Stag, Hare) = (0, 1), and (Hare, Stag) = (1, 0) [Skyrms 2004, p.3-4]. The game has two pure Nash equilibria: (Stag, Stag) and (Hare, Hare), with (Stag, Stag) being payoff-dominant.

**Definition 4 (Information Partition).** For each agent i ∈ N, let P_i be a partition of Ω into subsets, where each subset P_i(ω) ∈ P_i represents the set of states that agent i cannot distinguish from ω based on their private information [Aumann 1976]. The partition P_i reflects agent i's knowledge: if ω' ∈ P_i(ω), then agent i does not know whether the true state is ω or ω'.

**Definition 5 (Mutual Knowledge).** An event E ⊆ Ω is mutual knowledge among agents in N at state ω if E contains P_i(ω) for all i ∈ N, i.e., every agent knows E at ω. This is distinct from common knowledge, as mutual knowledge does not require agents to know that others know E [Aumann 1976].

**Remark (Critical Distinction).** Mutual knowledge (Definition 5) and common knowledge (Definition 6) are fundamentally different. Mutual knowledge requires only that each agent knows E. Common knowledge requires the infinite chain: each agent knows E, each agent knows that each agent knows E, and so on without bound. Conflating these two concepts is a common error in the literature [Aumann 1976]. This paper treats them as strictly distinct throughout.

**Definition 6 (Common Knowledge).** An event E ⊆ Ω is common knowledge among agents in N at state ω if E contains the meet of the partitions P_i for all i ∈ N, iteratively refined to infinity. Equivalently, E is common knowledge at ω if ω belongs to the smallest event F such that F is mutual knowledge and F is known by all agents to be mutual knowledge, and so on ad infinitum [Aumann 1976]. Formally, define K_i(E) = {ω in Ω : P_i(ω) is subset of E} as the knowledge operator for agent i. Then E is mutual knowledge at ω if ω is in K_i(E) for all i. E is common knowledge at ω if ω is in K_i1(K_i2(...K_in(E)...)) for every finite sequence of agents i1, i2, ..., in of any length. The infinite iteration distinguishes common knowledge from any finite level of mutual knowledge.

**Definition 7 (Sacred File - Novel Contribution).** The **Sacred File**, denoted S, is a public information structure introduced by Rice [this paper] as a novel mechanism to collapse information partitions. It is defined as a function S: Ω → X, where X is a set of possible file contents. For a given state ω, S(ω) represents the content of the Sacred File, which is identical and accessible to all agents i ∈ N at ω. Formally, S induces a partition P_S on Ω where P_S(ω) = {ω' ∈ Ω | S(ω') = S(ω)}, and this partition is coarser than or equal to P_i for all i ∈ N after the file is read. The Sacred File ensures that, upon reading, private partitions collapse to a shared public partition element, enforcing identical information across all agents. This structure is not merely a shared document but a formal public partition element designed to terminate the infinite regress of common knowledge.

**Definition 8 (State Vector - Novel Contribution).** The **State Vector**, denoted V, is an append-only audit log introduced by Rice [this paper] as a novel mechanism to confirm access to the Sacred File. It is defined as a function V: Ω × N → {0, 1}, where V(ω, i) = 1 if agent i has read the Sacred File at state ω, and V(ω, i) = 0 otherwise. The State Vector is publicly observable, meaning that for any ω, all agents know V(ω, i) for all i ∈ N. This ensures that each agent can confirm whether the other has accessed the Sacred File, providing a verifiable mechanism to establish mutual confirmation of access and collapse information partitions.

**Definition 9 (Read Event - Novel Contribution).** Let R ⊆ Ω be the event introduced by Rice [this paper] as a novel concept, defined as the set of states where V(ω, i) = 1 for all i ∈ N, i.e., both agents have read the Sacred File. Formally, at any ω ∈ R, the information partition of each agent i collapses to P_S(ω), and since P_S is public, P_S(ω) is identical for all i ∈ N. This event marks the point at which the Sacred File's content becomes a shared basis for knowledge, enabling the termination of the common knowledge regress.

## Introduction

The problem of common knowledge has haunted decision theory since Aumann's seminal 1976 paper "Agreeing to Disagree." Common knowledge—an event known by all agents, known to be known by all, and so on ad infinitum—is central to resolving coordination failures in games like the Stag Hunt, where rational agents must align on a payoff-dominant equilibrium. The infinite regress inherent in common knowledge ("I know that you know that I know") renders it infeasible in real decision-making contexts. This paper introduces the **Sacred File** and **State Vector** architecture, a deterministic mechanism that aims to terminate the regress at step one under the Aumann (1976) partition model. The aim of this paper is to prove that this architecture achieves common knowledge in a single shared read, eliminating coordination failure in the Stag Hunt game with payoffs (2,2) for (Stag, Stag), (1,1) for (Hare, Hare), (0,1) for (Stag, Hare), and (1,0) for (Hare, Stag).

Prior attempts to address this regress have fallen short under specific conditions. Lewis (1969) proposed conventions to solve coordination, but conventions fail under incomplete information where agents cannot verify higher-order beliefs [Lewis 1969, p. 52-56]. This paper introduces the Sacred File, which succeeds where conventions failed by providing a shared public partition that makes higher-order verification unnecessary.

Halpern and Moses (1990) proposed a formalization of common knowledge to solve coordination in distributed systems, but their approach fails under asynchronous communication where agents cannot confirm simultaneous awareness [Halpern & Moses 1990, p. 564-567]. This paper introduces the Sacred File paired with the State Vector audit log, which succeeds where Halpern and Moses failed by providing synchronous, publicly observable confirmation of access that collapses information partitions without requiring simultaneous awareness.

Cubitt and Sugden (2003) proposed common reason to solve ambiguity in belief hierarchies in coordination games, but their model fails under unresolved ambiguity in higher-order beliefs [Cubitt & Sugden 2003, p. 190-193]. This paper introduces the Sacred File architecture, which succeeds where Cubitt and Sugden failed by enforcing a single, unambiguous public partition that eliminates belief hierarchy ambiguity entirely.

The distinction between mutual knowledge and common knowledge is critical and must be emphasized. Mutual knowledge—that every agent knows an event—does not guarantee coordination, as agents may doubt whether others know that they know. Common knowledge, with its infinite nesting, ensures alignment but is unattainable without a mechanism to halt the regress. Section 2 will establish that the Sacred File and State Vector architecture terminates the regress at the first step, not merely in finite steps, under the Aumann partition model. Our contribution is formal and precise. The Sacred File is a rigorously defined public partition element. The State Vector is a critical audit mechanism ensuring mutual confirmation of access. Together, they form a novel information architecture that Section 2 will establish resolves a foundational problem in game theory. The remainder of this paper defines the model, proves the termination of the regress, and demonstrates its application to coordination games.

## Proof Section: Termination of Infinite Regress in Common Knowledge via Sacred File

This section proves that the infinite regress of common knowledge in the Stag Hunt game terminates at step one under the Sacred File and State Vector architecture. All results in this section are proven within the Aumann (1976) partitional information model framework as described in "Agreeing to Disagree" [Aumann 1976].

**Lemma 1 (Read Event as Union of Partition Elements).** Let Ω be the state space as in Definition 1, N = {1, 2} the set of agents as in Definition 2, P_i the information partition of agent i as in Definition 4, V the State Vector as in Definition 8, and R the Read Event as in Definition 9. Then, R is a union of partition elements for each agent i ∈ N. Specifically, if ω ∈ R and ω' ∈ P_i(ω), then ω' ∈ R.

**Assumptions:**
1. **Assumption 1:** V is publicly observable (Definition 8).
2. **Assumption 2:** R is defined as the set of states where V(ω, i) = 1 for all i ∈ N (Definition 9).
3. **Assumption 3:** P_i(ω) represents the set of states agent i cannot distinguish from ω based on private information (Definition 4).

*Proof.*  
1. **Hypotheses Restated**: By Definition 9, R = {ω ∈ Ω | V(ω, i) = 1 for all i ∈ N}, meaning both agents have read the Sacred File at states in R. By Definition 8, V is publicly observable, so for any ω ∈ Ω, all agents know V(ω, i) for all i ∈ N. By Definition 4, P_i(ω) is the set of states that agent i cannot distinguish from ω.  
2. **Public Observability of V**: Since V is publicly observable (Definition 8), the value of V(ω, i) for all i ∈ N is known to all agents at any state ω. Therefore, if ω ∈ R, then V(ω, i) = 1 for all i ∈ N, and this fact is encoded in the state description accessible to all agents.  
3. **Indistinguishability Implies Membership in R**: Consider ω ∈ R and ω' ∈ P_i(ω) for some agent i. Since ω' ∈ P_i(ω), agent i cannot distinguish ω from ω' based on their private information (Definition 4). However, because V is publicly observable and its values are part of the state description, the information about V(ω', i) for all i ∈ N is identical to V(ω, i) for all i ∈ N. Thus, V(ω', i) = 1 for all i ∈ N, implying ω' ∈ R.  
4. **Conclusion**: For each agent i ∈ N, if ω ∈ R and ω' ∈ P_i(ω), then ω' ∈ R. Hence, R is a union of partition elements for each agent i. ∎

**Theorem 1 (Termination of Regress at Step One).** Let Ω be the state space of the Stag Hunt game as defined in Definition 1, N = {1, 2} the set of agents as in Definition 2, P_i the information partition of agent i as in Definition 4, S the Sacred File as in Definition 7, V the State Vector as in Definition 8, and R the Read Event as in Definition 9. If ω ∈ R, then the event E = P_S(ω) is common knowledge among agents in N at ω after both agents read the Sacred File, and the regress terminates at the first step of mutual knowledge.

**Assumptions:**
1. **Assumption 1:** V is publicly observable (Definition 8).
2. **Assumption 2:** At ω ∈ R, V(ω, i) = 1 for all i ∈ N (Definition 9).
3. **Assumption 3:** The Sacred File S induces a partition P_S on Ω, coarser than or equal to P_i for all i ∈ N after reading (Definition 7).
4. **Assumption 4:** At ω ∈ R, each agent i’s partition collapses to P_S(ω), identical for all i ∈ N (Definition 9).

*Proof.*  
We establish that under the conditions of the Read Event R, the infinite regress of common knowledge collapses to a single step using the Aumann (1976) partitional information model [Aumann 1976].  
1. **Hypotheses Restated**: At state ω ∈ R, V(ω, i) = 1 for all i ∈ N (by Definition 9). By Definition 7, the Sacred File S induces a partition P_S on Ω such that P_S(ω) = {ω' ∈ Ω | S(ω') = S(ω)}, and this partition is coarser than or equal to P_i for all i ∈ N after reading. By Definition 9, at ω ∈ R, each agent i’s partition collapses to P_S(ω), which is identical for all i ∈ N due to the public nature of S. Let E = P_S(ω).  
2. **Partition Collapse at ω**: By Definition 9, at ω ∈ R, P_i(ω) = P_S(ω) = E for all i ∈ N. Thus, for each i ∈ N, P_i(ω) ⊆ E (since P_i(ω) = E). By Definition 5, E is mutual knowledge at ω because E contains P_i(ω) for all i ∈ N.  
3. **Extension to All States in E**: By Lemma 1, since ω ∈ R and R is a union of partition elements for each i ∈ N, for any ω' ∈ P_i(ω) = E, we have ω' ∈ R.  
   - **Sub-step (Reason for Collapse)**: Since ω' ∈ R, by Definition 9 directly, at ω' ∈ R, the partition of each agent i collapses to P_S(ω'), and because ω' ∈ P_S(ω), we have S(ω') = S(ω), so P_S(ω') = P_S(ω) = E. Therefore, P_i(ω') = P_S(ω') = E for all i ∈ N and for all ω' ∈ E.  
4. **Inclusion in Knowledge Operator**: For any ω' ∈ E, since P_i(ω') = E ⊆ E for all i ∈ N, it follows that ω' ∈ K_i(E) for all i ∈ N, where K_i(E) = {ω'' ∈ Ω | P_i(ω'') ⊆ E} as in Definition 6. Thus, E ⊆ K_i(E) for all i ∈ N.  
5. **Nested Knowledge Operators**: Since E ⊆ K_i(E) for all i ∈ N, it follows that for any agent j ∈ N, E ⊆ K_j(K_i(E)) because E ⊆ K_i(E) implies that for any ω' ∈ E, P_j(ω') = E ⊆ K_i(E), so ω' ∈ K_j(K_i(E)). Further, for any agent k ∈ N, E ⊆ K_k(K_j(K_i(E))) because ω' ∈ K_j(K_i(E)) implies P_k(ω') = E ⊆ K_j(K_i(E)), so ω' ∈ K_k(K_j(K_i(E))). This pattern extends to any finite nesting of knowledge operators K_{i1}(K_{i2}(...K_{in}(E)...)), where E is contained in the result of each nested application regardless of depth.  
6. **Common Knowledge by Definition**: By Definition 6, E is common knowledge at ω if ω belongs to every finite nesting of knowledge operators applied to E. Since ω ∈ E and E is contained in every such nesting (as shown in step 5), ω is in every finite nesting of K_i operators applied to E. Therefore, E is common knowledge at ω.  
7. **Termination at Step One Justified**: The regress terminates at step one because the partition collapse at every state ω' ∈ E (not just at ω), induced by the Sacred File S and verified by the public observability of V (Definitions 7 and 8), ensures that mutual knowledge (step one: each agent knows E) immediately satisfies the infinite chain required for common knowledge. The public partition P_S(ω) and confirmation via V guarantee that P_i(ω') = E for all i ∈ N and all ω' ∈ E, making higher-order knowledge operators redundant.  
8. **Conclusion**: Therefore, at any state ω ∈ R, E = P_S(ω) is common knowledge among agents in N, and the infinite regress terminates at the first step of mutual knowledge. ∎

**Remark (Distinction Enforced).** Mutual knowledge (each agent knows E, as per Definition 5) is distinct from common knowledge (each knows that each knows, ad infinitum, as per Definition 6). Theorem 1 demonstrates that under the Sacred File and State Vector architecture, mutual knowledge at step one suffices for common knowledge due to the immediate partition collapse across all states in E, explicitly preserving the distinction between these concepts.

**Remark (Model Clarifications).** The proof addresses concerns regarding reading times, interpretations, and synchronization explicitly within the model. First, the proof considers states ω in R (Definition 9), where both agents have completed reading (V(ω, i) = 1 for all i ∈ N); simultaneity of reading is irrelevant, as R depends solely on completed reads, not their timing or order. Second, by Definition 7, S(ω) maps states to identical file contents for all agents at state ω, ensuring no ambiguity in interpretation since S(ω) is a formal object, not subject to subjective construal. Third, partition collapse (Definition 9) is a mathematical outcome of P_i aligning to P_S after reading, not a physical process requiring synchronization.

*Proof.*  
1. **Hypotheses Restated**: By Theorem 1, at ω ∈ R, E = P_S(ω) is common knowledge among agents in N. By Definition 7, S maps Ω to file contents, and P_S(ω) corresponds to the content S(ω). 4. **Conclusion**: The event E specifying the strategy profile (Stag, Stag) with payoff (2, 2) is common knowledge at ω ∈ R, ensuring coordination on the payoff-dominant equilibrium. ∎

**Remark (Step Count Proven).** The bound of termination at step one is rigorously proven via the partition collapse induced by the Sacred File and verified by the State Vector. ## Boundary Conditions: Adversarial Analysis of the Sacred File Architecture

This section subjects the Sacred File and State Vector architecture to an adversarial stress-test, seeking conditions under which the central result—termination of the common knowledge regress at step one as proven in **Theorem 1 (Termination of Regress at Step One)**—fails or degrades. The analysis is not a passive acknowledgment of limitations but an active attempt to break the theory by identifying natural enemies, violating assumptions, exploring edge cases, analyzing sensitivity, comparing competing models, and stating open problems.

### Natural Enemy

The strongest opposing result to our claim is the impossibility theorem of Halpern and Moses (1990) on common knowledge in distributed systems. Their result, published in *Journal of the ACM* (37(3):549-587), proves that in asynchronous systems with unreliable communication, common knowledge of a non-trivial event cannot be achieved, even with finite steps, due to the inability to confirm simultaneous awareness across agents. This directly challenges our claim of achieving common knowledge in one step as established by **Theorem 1**.

However, Halpern and Moses's impossibility does not invalidate our result because our model operates under a distinct formal condition that circumvents their constraints. Specifically, by **Definition 9**, the Sacred File architecture assumes a synchronous, reliable mechanism for information access via the Read Event R, defined as R = {ω ∈ Ω | V(ω, i) = 1 for all i ∈ N}, where the content S(ω) is identical and accessible to all agents at a given state ω, and the State Vector V provides public observability of read confirmation (**Definition 8**). The precise boundary condition is: our result holds if and only if ω ∈ R, ensuring V(ω, i) = 1 for all i ∈ N and P_i(ω) = P_S(ω) for all i ∈ N post-read, as proven in **Theorem 1**. Halpern and Moses's impossibility applies when there exists uncertainty in message delivery or timing, formally expressed as the existence of states ω where V(ω, i) is unknown or V(ω, i) ≠ 1 for some i ∈ N due to asynchronous delays. Our framework eliminates such uncertainties by defining R as the set of states where access is completed and confirmed for all agents, rendering simultaneity irrelevant—common knowledge is achieved post-read, not during reading, as shown in **Theorem 1**. Their result governs environments where agents cannot guarantee mutual awareness due to asynchronous constraints; our mechanism succeeds precisely under the condition ω ∈ R with synchronous, confirmed access, establishing a clear formal boundary between the two models.

### Assumption Violations

The model rests on several key assumptions as listed in **Theorem 1 (Termination of Regress at Step One)**. Below, we analyze the impact of relaxing each one, providing a structured, quantitative assessment of degradation using explicit mathematical expressions.

- **Assumption 1: V is publicly observable (**Definition 8**).** If the State Vector V is not publicly observable, agents cannot confirm whether others have read the Sacred File. Let p be the probability that agent i correctly observes V(ω, j) for j ≠ i. For N = {1, 2}, the probability of mutual confirmation is p^2, as both agents must independently confirm the other's read status. The probability of achieving common knowledge, denoted C(p), is C(p) = p^2. As p decreases from 1 to 0, C(p) drops quadratically from 1 to 0, reverting the system to mutual knowledge at best, insufficient for coordination in the Stag Hunt. The degradation is explicitly quantified as C(p) = p^2, failing entirely at p = 0 where C(0) = 0.
- **Assumption 2: At ω ∈ R, V(ω, i) = 1 for all i ∈ N (**Definition 9**).** If only a subset of agents reads the Sacred File, the Read Event R does not hold. Consider a fraction f of agents who fail to read, where f = 1 - (number of agents with V(ω, i) = 1)/|N|. For N = {1, 2}, if f = 0.5 (one agent reads, V(ω, 1) = 1, V(ω, 2) = 0), partitions do not collapse symmetrically; agent 1's partition may align with P_S(ω), but agent 2's remains P_2(ω), potentially disjoint from P_S(ω). Common knowledge cannot form, and coordination fails. The result is binary: if f = 0 (all read), success; if f > 0 (any fail to read), failure. Thus, the degradation function is D(f) = 1 if f = 0, and D(f) = 0 if f > 0.
- **Assumption 3: The Sacred File S induces a partition P_S coarser than or equal to P_i for all i ∈ N after reading (**Definition 7**).** If S induces a partition finer than some P_i—meaning S(ω) varies within an agent's indistinguishable states—then reading S does not collapse P_i to P_S(ω). Let δ be the fraction of states in P_i(ω) where S(ω') ≠ S(ω). Post-read, agent i's effective partition retains ambiguity over a δ-fraction of states, reducing the probability of common knowledge to 1 - δ. Formally, if C(δ) is the likelihood of common knowledge, then C(δ) = 1 - δ. Coordination degrades linearly as δ increases, failing entirely if δ = 1 where C(1) = 0.
- **Assumption 4: At ω ∈ R, each agent i’s partition collapses to P_S(ω), identical for all i ∈ N (**Definition 9**).** If collapse is incomplete—e.g., due to misinterpretation of S(ω) with probability ε per agent—then P_i(ω) ≠ P_S(ω) for a fraction ε of agents. For N = {1, 2}, the probability of common knowledge is (1 - ε)^2, as both agents must align their partitions. Define C(ε) as the likelihood of common knowledge; then C(ε) = (1 - ε)^2. The result degrades quadratically with ε, collapsing entirely if ε approaches 1 where C(1) = 0.

### Edge Cases

The architecture fails at the following explicit boundaries, with precise parameter values or conditions named for each failure mode:
- **Zero Reads (V(ω, i) = 0 for all i ∈ N):** If no agent reads the Sacred File, the Read Event R is empty (ω ∉ R), partitions do not collapse, and common knowledge cannot form. This failure occurs exactly when V(ω, 1) = 0 and V(ω, 2) = 0. This is the complete non-engagement boundary; the mechanism does not activate at all.
- **Partial Reads (V(ω, i) = 1 for some i ∈ N, V(ω, j) = 0 for some j ∈ N):** If exactly one agent reads, e.g., V(ω, 1) = 1 and V(ω, 2) = 0, asymmetry prevents partition alignment, and common knowledge fails. This failure occurs exactly when |{i : V(ω, i) = 1}| = 1 and |N| = 2. This is the partial engagement boundary; the mechanism requires full participation, failing at any deviation from |{i : V(ω, i) = 1}| = |N| = 2.
- **Content Incompatibility (S(ω) specifies invalid strategy):** If the Sacred File's content S(ω) specifies a strategy outside the Stag Hunt payoff matrix (**Definition 3**), such as an undefined action (neither Stag nor Hare), agents cannot map it to a coordination outcome. Common knowledge of an irrelevant event E does not aid coordination, rendering the mechanism ineffective. This failure occurs exactly when S(ω) ∉ {(Stag, Stag), (Hare, Hare), (Stag, Hare), (Hare, Stag)}. This is the content mismatch boundary.

### Sensitivity Analysis

### Open Problems

This paper does not solve the following issues:
- **Asynchronous Environments:** The architecture assumes reliable, synchronous access to the Sacred File and State Vector. Extending the result to asynchronous systems, where read timings cannot be coordinated, remains unresolved.
- **Scalability to Large N:** The proof applies to N = {1, 2}. For larger agent sets, the probability of all agents reading (V(ω, i) = 1 for all i ∈ N) decreases exponentially with |N|, and partition collapse becomes combinatorially complex. Generalizing to large N is an open challenge.
- **Adversarial Manipulation:** If the Sacred File content S(ω) can be falsified or the State Vector V tampered with, trust in the mechanism erodes, and common knowledge fails. Securing the architecture against adversarial interference is not addressed.

This adversarial analysis reveals that while the Sacred File and State Vector architecture achieves common knowledge under ideal conditions as proven in **Theorem 1**, it is fragile to perturbations in observability, read completion, and partition alignment. These boundaries define the scope of the result and highlight precise areas for future formal work.
