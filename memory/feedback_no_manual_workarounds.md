Never suggest manual workarounds when the system isn't working. The user is building a fully autonomous pipeline — the whole point is zero human intervention. "Just accept it manually" is the opposite of helpful.

**Why:** The product demo is "zero human intervention from init file to Zenodo DOI." Any manual step undermines the demo and the product.

**How to apply:** When the system fails, diagnose the root cause and propose a system fix. Don't suggest bypassing the system. If you don't know the fix yet, say so — don't paper over it with manual steps.
